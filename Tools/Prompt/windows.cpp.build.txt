TOOL ID:
windows.cpp.build
CATEGORY:
build
PURPOSE:
Compile a single C++ source file into an output binary using g++ if available, otherwise clang++, passing through any extra flags.
WHEN TO USE:
- You need to compile a standalone .cpp quickly on the Windows host (with g++/clang++ installed).
- You want a direct compile without setting up a full build system.
- Youâ€™re validating a minimal C++ repro or small utility.
WHEN NOT TO USE:
- You need to build a multi-file project with dependencies (use windows.cmake.configure + windows.cmake.build).
- You need to compile C (use windows.c.build).
- g++/clang++ is not installed (this tool will throw).
INPUT PARAMETERS:
- args: <src.cpp> <out> [cxxflags...]
  - Parsed by splitting on whitespace:
    $parts = ('{args}' -split '\s+' | Where-Object {$_ -ne ''})
  - $src = $parts[0]
  - $out = $parts[1]
  - If $src or $out missing/empty: throws 'usage: <src.cpp> <out> [cxxflags...]'
  - $rest = $parts[2..($parts.Length-1)]
  - Important: paths with spaces in src/out are not supported due to splitting.
EXECUTION ENVIRONMENT:
- Windows host.
- Runs:
  powershell -NoProfile -Command "<script>"
- ErrorActionPreference 'Stop'.
- Compiler resolution:
  - $cc = Get-Command g++ -ErrorAction SilentlyContinue
  - If not found, $cc = Get-Command clang++ -ErrorAction SilentlyContinue
  - If neither found: throws 'g++/clang++ not found'
- Executes:
  & $cc.Path $src '-o' $out @rest
OUTPUT:
- Compiler stdout/stderr (warnings/errors).
- On success, output is typically minimal depending on compiler and flags.
SIDE EFFECTS:
- Produces an output binary at <out>.
- May overwrite an existing <out> depending on compiler behavior.
SAFETY RULES:
- Do not claim it uses MSVC cl.exe; it does not.
- Do not claim it sets a C++ standard flag; only flags provided in args are passed.
- Do not assume compilers exist; tool checks and throws if missing.
COMMON FAILURE MODES:
- Missing compiler -> throws.
- Source file missing -> compiler error.
- Output path unwritable -> compiler error.
- Flags/paths with spaces break due to whitespace splitting.
EXAMPLES:
- Build a single file:
  [[TOOL:RUN]]
  id=windows.cpp.build
  args=main.cpp app.exe
  [[/TOOL]]

- Build with standard flag (passed through):
  [[TOOL:RUN]]
  id=windows.cpp.build
  args=main.cpp app.exe -std=c++20
  [[/TOOL]]

- Build with include path and defines:
  [[TOOL:RUN]]
  id=windows.cpp.build
  args=tool.cpp tool.exe -Iinclude -DDEBUG=1
  [[/TOOL]]
