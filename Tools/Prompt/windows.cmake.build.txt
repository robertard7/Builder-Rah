TOOL ID:
windows.cmake.build
CATEGORY:
build
PURPOSE:
Build a previously configured CMake build directory using `cmake --build <buildDir>` optionally with `--target <target>`.
WHEN TO USE:
- You have already configured a CMake project and want to compile it.
- You want to build the default target(s) or a specific named target.
- You are using the same generator chosen during configure (this tool does not set generator).
WHEN NOT TO USE:
- You have not configured the build directory (run windows.cmake.configure first).
- CMake is not installed (tool throws).
- You need to run tests (not covered here).
INPUT PARAMETERS:
- args: <buildDir> [target]
  - Parsed by splitting on whitespace:
    $parts = ('{args}' -split '\s+' | Where-Object {$_ -ne ''})
  - $b = $parts[0]
  - $t = $parts[1]
  - If $b missing/empty: throws 'usage: <buildDir> [target]'
  - target token is optional; if omitted, builds default targets.
  - Important: buildDir with spaces is not supported due to splitting.
EXECUTION ENVIRONMENT:
- Windows host.
- Runs:
  powershell -NoProfile -Command "<script>"
- ErrorActionPreference 'Stop'.
- Checks:
  if(!(Get-Command cmake -ErrorAction SilentlyContinue)) { throw 'cmake not found' }
- Executes:
  - If $t is empty:
    cmake --build $b
  - Else:
    cmake --build $b --target $t
OUTPUT:
- Build output from the underlying build tool selected by CMake (often Ninja if configured that way).
SIDE EFFECTS:
- Produces build artifacts inside the build directory.
- May rebuild incrementally depending on build system state.
SAFETY RULES:
- Do not claim it configures or regenerates; it only builds.
- Do not claim it sets parallelism flags; it does not.
- Do not claim it sets configuration (Debug/Release) unless configured beforehand or passed by CMake config.
COMMON FAILURE MODES:
- cmake missing -> throws.
- buildDir invalid or not configured -> build errors.
- Underlying generator/build tool missing (e.g., Ninja not installed) -> build fails.
- Target name wrong -> build fails.
EXAMPLES:
- Build default target(s):
  [[TOOL:RUN]]
  id=windows.cmake.build
  args=.\build
  [[/TOOL]]

- Build a specific target:
  [[TOOL:RUN]]
  id=windows.cmake.build
  args=.\build MyTarget
  [[/TOOL]]

- Typical flow:
  1) windows.cmake.configure .\src .\build
  2) windows.cmake.build .\build
