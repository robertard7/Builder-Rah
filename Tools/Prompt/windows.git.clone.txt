TOOL ID:
windows.git.clone
CATEGORY:
git
PURPOSE:
Clone a git repository into the current directory using `git clone <url> <dest>`.
WHEN TO USE:
- You need to fetch a repo locally for building or inspection.
- You want to clone into a specific folder name (dest) under the current directory.
WHEN NOT TO USE:
- The destination already exists and you need to update it (use git pull manually via windows.run.cmd, not provided as a dedicated tool here).
- You need shallow clone or branch selection (not supported by this tool).
- You need to clone into a path containing spaces (arg splitting will likely break).
INPUT PARAMETERS:
- args: <url> <dest>
  - Parsed by splitting on whitespace:
    $parts = ('{args}' -split '\s+' | Where-Object {$_ -ne ''})
  - $url = $parts[0]
  - $dest = $parts[1]
  - If either is missing/empty, throws: 'usage: <url> <dest>'
  - Any additional tokens beyond the first two are ignored (not used by the command).
  - Important: URLs or destinations containing spaces cannot be represented reliably.
EXECUTION ENVIRONMENT:
- Windows host.
- Runs:
  powershell -NoProfile -Command "<script>"
- ErrorActionPreference 'Stop'.
- Uses:
  git clone $url $dest
- Requires `git` in PATH.
OUTPUT:
- Raw stdout/stderr from git clone (progress, messages).
SIDE EFFECTS:
- Creates a new directory named <dest> in the current directory.
- Downloads repository objects from the remote URL.
SAFETY RULES:
- Do not claim it checks out a specific branch or depth; it performs a default clone.
- Do not claim it validates URL beyond gitâ€™s behavior.
- Cloning from untrusted URLs can introduce unsafe code; treat new repos as untrusted until reviewed.
COMMON FAILURE MODES:
- git not installed.
- Network/auth issues (private repos, blocked TLS, proxy).
- Destination folder already exists and is not empty -> git clone fails.
- URL invalid or unreachable.
EXAMPLES:
- Clone a public repo:
  [[TOOL:RUN]]
  id=windows.git.clone
  args=https://github.com/example/project.git project
  [[/TOOL]]

- Clone into a specific folder name:
  [[TOOL:RUN]]
  id=windows.git.clone
  args=https://github.com/rah/rah.git RahRepo
  [[/TOOL]]

- After clone:
  1) windows.repo.tree RahRepo
  2) windows.git.status RahRepo
