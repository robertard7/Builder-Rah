TOOL ID:
windows.repo.finddir
CATEGORY:
repo
PURPOSE:
List directories under the current directory recursively to a specified depth (default 4), returning up to 600 directory FullName values.
WHEN TO USE:
- You want a directory-only inventory to locate project folders.
- You need a bounded recursive scan with controllable depth.
- Youâ€™re finding where a repo/project lives under the current working directory.
WHEN NOT TO USE:
- You want files too (use windows.repo.tree with tree command, or windows.file.find).
- You need to search inside files (use windows.grep.*).
- You need a single directory listing (use windows.repo.ls).
INPUT PARAMETERS:
- args: <maxdepth optional>
  - Entire raw args assigned to $d = '{args}'
  - If args is null/empty/whitespace, defaults to 4
  - If args does not match ^[0-9]+$, defaults to 4
  - Uses Depth ([int]$d) for Get-ChildItem -Recurse -Directory -Depth
- Note: This tool uses current directory as root; it does not accept a path argument in the manifest.
EXECUTION ENVIRONMENT:
- Windows host.
- Runs:
  powershell -NoProfile -Command "<script>"
- ErrorActionPreference 'Stop'.
- Uses:
  Get-ChildItem -Recurse -Directory -Depth ([int]$d) -Force
  | Select-Object -First 600 FullName
OUTPUT:
- Up to 600 directory FullName strings printed to stdout.
SIDE EFFECTS:
- None (read-only).
SAFETY RULES:
- Depth recursion can be expensive in large trees; keep depth small when possible.
- Do not assume -Depth is supported on all PowerShell versions; if it errors, it errors.
COMMON FAILURE MODES:
- -Depth parameter not supported (depends on PowerShell version).
- Access denied to some directories; may error out because ErrorActionPreference is Stop.
- Very large directory trees may be slow.
- Passing non-numeric args triggers default 4 (not an error).
EXAMPLES:
- Default depth (4):
  [[TOOL:RUN]]
  id=windows.repo.finddir
  [[/TOOL]]

- Depth 2:
  [[TOOL:RUN]]
  id=windows.repo.finddir
  args=2
  [[/TOOL]]

- Bad args falls back to 4:
  [[TOOL:RUN]]
  id=windows.repo.finddir
  args=two
  [[/TOOL]]

ANTI-HALLUCINATION NOTES:
- Only directories are listed, not files.
- Root is the current directory only; you cannot specify a different base path with this tool.
- Output is capped to first 600 results; missing directories may simply be beyond the cap.
- Non-numeric args become 4 silently; do not claim it used the provided value.
- Do not claim it returns relative paths; it returns FullName (absolute or resolved) as provided by PowerShell.
